Detailed Description

RendezvousSession establishes and maintains the first connection between a commissioner and a device.

This connection is used in order to provide the necessary infos for a device to participate to the CHIP ecosystem.

All the information transmitted over the underlying transport are encrypted upon establishment of an initial secure pairing session.

In order to securely transmit the informations, RendezvousSession requires a setupPINCode to be shared between both ends. The setupPINCode can be configured using RendezvousParameters

The state of the secure pairing session setup can be observed by passing a RendezvousSessionDelegate object to RendezvousSession. Both the commissioner and the device needs to bootstrap RendezvousSession using RendezvousParameters.
